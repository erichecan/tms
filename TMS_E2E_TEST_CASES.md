# TMS 系统端到端（E2E）测试用例

> 本文档覆盖系统全部功能模块，每条测试用例体现**完整业务流**。

---

## 测试 1：运单完整生命周期（Default 模板 - 端到端）

1. 登录系统（Admin 账号），验证仪表盘正常展示
2. 进入「客户管理」→ 新增客户：公司名=TestCorp，联系人=张三，电话=4371234567，邮箱=test@corp.com，地址=1399 Kennedy Rd，信用额度=50000，状态=Active
3. 验证：客户列表出现 TestCorp 行，状态为 Active ✅
4. 进入「运单列表」→ 点击「创建运单」
5. 选择模板 = **Default**
6. 填写发货方：公司=Apony Group, 地址=1399 Kennedy Road（通过 Google Autocomplete）
7. 填写收货方：公司=TestCorp, 地址=100 Queen St W, Toronto
8. 在 Cargo Manifest 中添加 2 条货物行：
   - 行1：Pallets=5，Items=200，PRO=PRO-001，PO=PO-2026-001
   - 行2：Pallets=3，Items=150，PRO=PRO-002，PO=PO-2026-002
9. 选择 Client = TestCorp
10. 验证：系统自动计算 Price（基于 Google Maps 距离和规则引擎）
11. 绘制电子签名
12. 点击保存
13. 验证：页面跳转到运单列表，新运单状态 = **NEW** ✅
14. 在搜索框输入运单号，验证搜索结果能找到该运单 ✅
15. 点击运单操作菜单 → **指派（Assign）**
16. 弹出指派模态框，选择 司机 + 车辆，点击确认
17. 验证：运单状态变为 **ASSIGNED** ✅
18. 进入「财务管理 → 应收账款」
19. 验证：TestCorp 下出现一条新的应收记录，金额 = 运单估价 ✅
20. 进入「财务管理 → 应付账款」
21. 验证：对应司机下出现一条应付记录（driver_pay） ✅
22. 回到运单列表，将运单标记为 **DELIVERED**
23. 验证：状态更新为 DELIVERED ✅
24. 进入「财务概览」
25. 验证：Revenue、Expenses、Profit 数值正确反映本次运单 ✅

## 测试 2：运单完整生命周期（Amazon 模板 - 端到端）

1. 登录系统
2. 进入「运单列表」→ 创建运单
3. 切换模板为 **Amazon**
4. 在 Shipment Appointment 区域粘贴/拖拽 ISA 截图、Barcode 截图
5. 填写 Base Info：
   - Fulfillment Center Alias = Y001
   - Reference Code = APT-2026-001
   - Delivery Date = 明天, Time = 09:00
   - FC Address = 8050 Heritage Rd, Brampton
6. 在 Amazon Manifest 表格中添加 3 条货物：
   - 行1：PRO=PRO-A1, BOL Vendor Ref=VR-001, Vendor=AcmeCo, Pallets=4, Cartons=40, Units=400, PO=PO-A1
   - 行2：PRO=PRO-A2, BOL Vendor Ref=VR-002, Vendor=WidgetInc, Pallets=2, Cartons=20, Units=200, PO=PO-A2
   - 行3：PRO=PRO-A3, BOL Vendor Ref=VR-003, Vendor=GlobalLtd, Pallets=6, Cartons=60, Units=600, PO=PO-A3
7. 填写 Client Name, Distance, Price
8. 点击保存
9. 验证：运单创建成功，状态 = NEW ✅
10. 打开运单查看模式（View），验证 ISA 图片、Barcode 图片都正确展示 ✅
11. 打开运单编辑模式（Edit），修改 Delivery Date 为后天
12. 保存，验证：运单更新成功，日期已变更 ✅
13. 执行指派流程（选司机 + 车辆）
14. 验证：运单状态 → ASSIGNED ✅
