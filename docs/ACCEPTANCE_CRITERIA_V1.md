# 正式上线验收标准 v1.0

## 评分模型

| 维度 | 权重 | 验收要点 | 评分说明 |
| --- | --- | --- | --- |
| 产品与流程完备性 | 25 | 运单→调度→司机→POD→财务全链路 3 套真实案例跑通；移动端司机可独立操作；客户自助创建/查询可用 | 每缺一关键流程 -5 分；移动端缺失 -10 分；客户入口缺失 -5 分 |
| 数据准确性与幂等 | 20 | 数据迁移脚本可一键执行且含校验；运单/财务/规则类接口支持 Idempotency；审计日志全量落库 | 迁移校验缺失 -10 分；每项幂等缺失 -5 分；审计覆盖<90% -5 分 |
| 性能与稳定性 | 15 | Cloud Run 压测 P95<500ms、错误率<0.5%；调度任务持续 24h 无异常；位置更新 5s 内可见 | 未压测 -10 分；指标不达标按比例扣分；调度异常 -5 分 |
| 安全与权限 | 15 | JWT 校验无绕过；RBAC 覆盖关键模块；敏感字段脱敏；Secrets 受控 | 开发绕过存在 -15 分；RBAC 缺口 -5 分；脱敏缺失 -5 分；Secrets 不合规 -5 分 |
| 运维与监控 | 15 | Prometheus/Grafana/Alertmanager 上线；日志集中化；每日备份+恢复演练 | 每缺一项 -5 分；无恢复演练 -5 分 |
| 文档与交付物 | 10 | 发布手册、回滚方案、FAQ、残留风险清单齐备；测试报告覆盖≥85% | 缺任一文档 -5 分；测试覆盖不足 -5 分 |

## 验收流程

1. <!-- 2025-11-11T15:05:12Z Added by Assistant: Acceptance preparation steps -->准备材料：负责团队在验收前 2 天提交测试报告、风险清单、监控截图。
2. <!-- 2025-11-11T15:05:12Z Added by Assistant: Acceptance committee description -->验收小组：产品负责人、技术负责人、运维负责人组成，必要时邀请安全与财务代表列席。
3. <!-- 2025-11-11T15:05:12Z Added by Assistant: Scoring process -->评分会议：依照上表逐项打分并记录证据链接。
4. <!-- 2025-11-11T15:05:12Z Added by Assistant: Threshold enforcement -->门槛：总分≥80 且无阻塞级风险方可进入发布窗口；若未达标需生成整改单。
5. <!-- 2025-11-11T15:05:12Z Added by Assistant: Sign-off rule -->签字：各负责人在验收记录上电子签字确认，存档于 `docs/release/`。
