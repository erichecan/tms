# TMS 主流程设计文档

**版本：** 1.0  
**日期：** 2026-04-11  
**状态：** 已确认  
**业务场景：** 北美专线运力（FBA 头程 + 代发 + 转运）

---

## 一、目标

本文档定义 TMS 系统的"一单到底"主流程剧本，覆盖以下四个用途：

1. **手工测试剧本** — 逐步操作验证系统功能
2. **员工培训手册** — 各角色了解自己的操作流程
3. **演示脚本** — 向客户/管理层展示系统能力
4. **E2E 自动化测试参考** — 对应每个自动化测试步骤

- **取货地址：** 100 Steeles Ave W, Toronto, ON M3J 2Z2（Sunny Trade 仓库）
- **送达地址：** 8050 Heritage Rd, Brampton, ON L6Y 0R9（Amazon YYZ9）

## 四、流程总览

```
阶段 1  [Admin Eric]          初始配置：账号 + 计费规则 + 合作报价 + 司机薪酬
         ↓
阶段 2  [Dispatcher 张伟]     接到陈小明电话 → 手动创建转运单 → 自动生成运单
         ↓
阶段 3  [Dispatcher 张伟]     运单管理 → 智能推荐 + 手动可改 → 指派王建国 + TRK-001
                               → 系统创建行程（Trip）
         ↓
阶段 4  [Driver 王建国]       手机端确认行程 → GPS 上报位置 → 到仓库取货
                               → 系统记录 Time In
         ↓
阶段 5  [Driver 王建国]       运往 YYZ9 → 电子签收 → 上传 POD 照片
                               → 确认完成，系统记录 Time Out
         ↓
阶段 6  [System 自动]         计费引擎触发：计算应收（客户）+ 应付（司机工资）
         ↓
阶段 7  [Finance 李梅]        审核应收/应付 → 确认结算 → 标记已付款
        [CEO Eric]            Dashboard 查看全局数据、地图轨迹、财务汇总
        [车队经理 赵强]       查看司机行程效率、Time In/Out 记录
```

**账号：** `admin`

#### 1.2 配置计费规则

进入**规则管理 → 新建规则**：

- 规则名称：`FBA头程标准计费`
- 计费方式：按托盘（Per Pallet）
- 单价：$45.00 CAD / pallet
- 燃油附加费：8%（自动叠加）
- 适用线路：Toronto → YYZ9

#### 1.3 配置合作报价卡

进入**规则管理 → 合作单位报价**（或 Partner Pricing）：

- 客户：Sunny Trade
- 绑定规则：`FBA头程标准计费`
- 信用额度：$50,000

#### 1.4 配置司机薪酬

进入**车队管理 → 司机档案 → 王建国**：

- 薪酬类型：提成制
- 提成比例：25%（按运单应收金额）

**预期结果：** 所有账号可正常登录；规则管理页可查到新规则；王建国档案显示薪酬配置。

### 阶段 6 — 系统自动计费

> 无需人工操作，系统在运单变为 `Delivered` 后自动执行。

| 计算项 | 计算逻辑 | 结果 |
|--------|---------|------|
| **基础运费** | 100 pallets × $45.00 | $4,500.00 |
| **燃油附加费** | $4,500 × 8% | $360.00 |
| **应收总额（陈小明）** | $4,500 + $360 | **$4,860.00 CAD** |
| **应付（王建国工资）** | $4,860 × 25% | **$1,215.00 CAD** |
| **公司毛利** | $4,860 - $1,215 | **$3,645.00 CAD** |

**预期结果：**
- 财务应收列表出现 Sunny Trade 这笔 $4,860 的待收款
- 财务应付列表出现王建国这笔 $1,215 的待付工资
- 运单详情费用明细栏显示完整计费分解

## 七、异常场景（扩展测试）

主流程跑通后，可进一步验证以下异常场景：

| 场景 | 触发方式 | 预期系统行为 |
|------|---------|------------|
| 改派司机 | 张伟在运单 `Assigned` 状态下重新指派 | 原司机通知取消，新司机收到通知 |
| 拆分运单 | 运单详情 → 点击 Split | 生成两张子运单，分别可独立分配 |
| 标记异常 | 行程中点击"标记异常" | 运单状态变 Exception，调度收到告警 |
| 异常恢复 | 张伟处理后点击"恢复行程" | 状态恢复 In Transit，继续执行 |
| 取消运单 | `Assigned` 状态前取消 | 司机释放，运单变 Canceled |
