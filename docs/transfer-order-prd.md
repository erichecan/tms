## 转运单创建与托盘管理 PRD — 2026-03-13 20:32:00

> 版本：v0.1（草案，用于驱动第一版实现）  
> 范围：仅覆盖“运营手动录入转运单 → 明细/HOLD 管理 → 生成运单/Trip”的能力，与报价卡对接仅做最小可用。

---

### 1. 背景与目标

#### 1.1 背景

- 现有 TMS 已支持：运单（Waybill）、行程（Trip）、司机端操作、财务结算等功能，并有完整 API 与集成测试。
- 实际业务中，转运/派送业务长期通过 Excel 记录（如《2026年转运汇总表.xlsx》），包含：
  - 柜号、所在仓库、进仓方式、到仓日期；
  - 分货 SKU、FBA、PoList、件/箱数、托数、方数、目的仓；
  - HOLD / 暂不处理 等业务备注。
- 各合作单位（JW、ESC、无忧达、Andy 等）的价格规则记录在《合作单位报价卡.xlsx》中。
- 当前系统中 **没有“转运单 + 多行明细 + 托盘级 HOLD 管理”这一层业务模型和界面**，运营只能直接从“运单/Trip”维度往后看，无法在系统内完成真实的转运录入与编排。

#### 1.2 目标

本 PRD 期望在现有 TMS 上新增一层“转运单”能力，使业务起点从：

- “Excel 里新增一行转运记录”

升级为：

- **运营同事在 TMS 中创建一张“转运单”，并管理其下的货物/托盘明细**，进而：
  - 标记某些托盘为 HOLD，记录 hold 仓库与原因；
  - 按需批量生成运单（Waybills）并驱动后续 Trip/司机/财务链路；
  - 为后续自动化对接报价卡打基础。

---

### 2. 概念与范围

#### 2.1 核心概念

- **转运单（Transfer Order）**
  - 描述“一票转运需求”的头信息；
  - 例如：某柜号的货物在某仓库，于某日到仓，需要向多个目的仓派送。

- **转运单明细（Transfer Order Line / Pallet Item）**
  - 对应一组货物的明细记录（通常以托盘或 SKU+PoList 粒度）；
  - 一张转运单下可以有多行明细。

- **HOLD 托盘**
  - 处于暂不出货状态的明细记录（全部或部分托盘）；
  - 常见原因：客户要求暂缓、价格/结算未确定（如 RMB 结算暂不处理）、等待与其他柜合并等。

- **运单（Waybill）**
  - 系统已有概念；
  - 由一条或多条“转运单明细”汇总生成；
  - 需要能反查来源：哪些转运单/明细/托盘贡献了本运单。

#### 2.2 本期范围（v0.1）

**纳入范围：**

- 新增“转运单”头部 + 明细的数据模型与后端 API；
- 后台 UI 中新增：
  - 转运单列表页；
  - 转运单详情/编辑页，支持多行明细录入；
  - 明细级 HOLD 标记/解除能力；
  - 从勾选明细生成运单（单张 / 多张）能力；
- 与现有运单/Trip 功能的最小联动：
  - 从转运单明细生成运单；
  - 运单记录来源（转运单 ID + 明细 ID）。

**暂不纳入范围（可留空/保留字段）：**

- 自动从 Excel 批量导入形成转运单；
- 报价卡在线算价的完整规则引擎（本期仅设计字段结构和最小钩子）；
- 司机端对托盘级别的实时操作（司机仍以运单/Trip 为主）。

---

### 3. 角色与权限

- **运营/调度（Dispatcher / R-DISPATCHER）**
  - 可新建/编辑/查看转运单；
  - 可编辑转运单明细；
  - 可标记/解除 HOLD；
  - 可从明细生成运单（含批量）。

- **管理员（Admin / R-ADMIN）**
  - 拥有运营所有权限；
  - 额外可删除/取消转运单（如误建）。

- **司机 / 普通用户**
  - 本期不开放转运单相关入口。

---

### 4. 数据结构与字段（产品视角）

#### 4.1 转运单头（Transfer Order Header）

**必填字段：**

- `转运单号`：系统自动生成（例：`TO-20260313-0001`）。
- `客户/业务方`：下拉或自动补全，关联 `customers`。
- `合作单位`：下拉，来源于配置（如 JW、ESC、无忧达、Andy 等）。
- `柜号/提单号`：字符串。
- `所在仓库`：如 `7 仓`、`25 仓`、`Frank 仓`，下拉。
- `进仓方式`：枚举（`整柜` / `散货` / `散板`）。
- `到仓日期`：日期。

**选填字段：**

- `主目的仓`：下拉（`YYZ3/YYZ4/YYZ7/YOW1/YHM1` 等），可为空；
- `币种`：枚举（`CAD` / `USD` / `RMB`）；
- `整体备注`：多行文本。

**系统字段：**

- `状态`：枚举：
  - `DRAFT`：草稿；
  - `PARTIAL_WAYBILLED`：部分明细已生成运单；
  - `COMPLETED`：所有明细已生成或取消；
  - `CANCELLED`：整单作废。
- `创建人 / 创建时间`；
- `最后修改人 / 最后修改时间`。

#### 4.2 转运单明细（Transfer Order Line）

**基础信息：**

- `行号`：系统生成。
- `SKU/分货 SKU`：字符串。
- `FBA 号`：字符串。
- `PoList`：字符串。
- `柜号`：冗余（自动带出自头部，可覆盖）。
- `所在仓库`：冗余，支持细化到库位（可选）。

**数量与规格：**

- `件/箱数`：数值。
- `托数（Pallets）`：数值（可为小数，例如半托）。
- `方数（CBM）`：数值。
- `重量（kg）`：可选。

**去向与合作信息：**

- `目的仓`：下拉（`YYZ3/YYZ4/YOW1/YHM1/...`）。
- `派送类型`：枚举（`直卡` / `拖车` / `散板` / `拼车`）。
- `合作单位`：默认继承自头部，可行级覆盖。
- `预计发车日期`：日期，可空。

**计费相关（预留字段）**：

- `价格模式`：枚举（`直卡` / `拖车` / `散板/拼货` / `一票一议`）。
- `建议客户报价`：数值（由规则引擎/报价卡匹配得出，可覆盖）。
- `建议成本价`：数值。
- `手工调整金额`：数值（可为正/负）。
- `调整原因`：文本。

**HOLD 管理：**

- `hold 状态`：枚举：
  - `NORMAL`：正常；
  - `HOLD_PENDING`：暂不处理；
  - `HOLD_LONGTERM`：长期 hold；
  - `RELEASED`：曾 hold 现已释放。
- `hold 仓库`：下拉（与所在仓库列表一致）；
- `hold 原因`：文本（例如“RMB 结算暂不处理”、“客户要求暂缓”等）；
- `预计释放日期`：日期，可空。

**生成运单关联：**

- `已生成托数` / `已生成方数`：数值；
- `剩余可用托数` / `剩余可用方数`：数值；
- `关联运单列表`：只读，显示关联的 Waybill 编号集合。

### 5. 关键业务流程

#### 5.1 创建转运单（运营录入）

1. 运营在后台菜单进入：`转运管理 → 转运单列表`。
2. 点击“新建转运单”：
   - 填写头部字段（柜号、所在仓、到仓日、客户、合作单位等）。
3. 在“明细列表”区域：
   - 通过表格逐行录入明细（支持复制粘贴 Excel 的多行内容）；
   - 可为每一行设置：
     - 分货 SKU / FBA / PoList；
     - 件数、托数、方数；
     - 目的仓、派送类型；
     - 是否 HOLD 及 hold 仓库/原因。
4. 运营可随时点击：
   - “保存草稿”：状态置为 `DRAFT`，可日后继续编辑；
   - 或“保存并准备生成运单”：仍保持 `DRAFT`，但引导进入下一步操作。

#### 5.2 HOLD 托盘管理

**创建 / 标记：**

- 在录入或编辑明细时，运营可以：
  - 勾选“标记为 HOLD”；
  - 必填：`hold 仓库` 与 `hold 原因`；
  - 系统将明细行状态设为 `HOLD_PENDING` 或 `HOLD_LONGTERM`。

**HOLD 的约束：**

- 处于 HOLD 状态的托数/方数：
  - **不能被纳入“生成运单”的选择集合**；
  - 在明细表中以醒目的颜色/图标标识。

**部分 HOLD 的场景：**

- 若一行有 10 托，其中 3 托需要 HOLD：
  - 产品允许运营通过“拆分明细”操作，将一行拆为两行：
    - 行 A：7 托，`hold 状态 = NORMAL`；
    - 行 B：3 托，`hold 状态 = HOLD_PENDING`。

**解除 HOLD：**

- 当业务条件满足（客户确认、结算方式明确等），运营可：
  - 在明细行操作中选择“解除 HOLD”；
  - 选择新的 `hold 状态 = RELEASED` 或完全恢复为 `NORMAL`。
- 解除后，相关托盘重新出现在可选集合中，可与其他转运单明细一起生成新的运单。

#### 5.3 从转运单生成运单（单票/批量）

**单票生成：**

1. 在转运单详情页，运营勾选若干条 **非 HOLD 且有剩余托盘/方数** 的明细。
2. 点击“生成运单”按钮，打开预览弹窗：
   - 弹窗显示：
     - 合作单位、始发仓、目的仓；
     - 选中明细总托数/总方数；
     - 建议客户报价 / 成本价（如有）。
   - 运营可：
     - 微调价格；
     - 填写运单级别的备注（如装卸要求、时间窗口等）。
3. 确认后：
   - 系统调用现有 `POST /api/waybills` 接口；
   - 在运单的 details/扩展字段中写入来源信息：
     - `transfer_sources: [{ transferOrderId, lineId, pallets, cbm }]`；
   - 更新对应明细行的：
     - `已生成托数/方数` 与 `剩余可用托数/方数`；
     - 若某行全部数量都已生成，标记该行为“已全部生成”。

**批量生成：**

- 在转运单详情页或列表页，支持：
  - 跨多张转运单勾选多条明细；
  - 按“合作单位 + 目的仓 + 发车日期”自动分组：
    - 每一组生成一张运单；
    - 在预览界面中展示分组结果；
    - 允许运营手动调整（合并/拆分某些组）。

**与 Trip 的衔接：**

- 本期仅要求保障：
  - 运单一旦创建，即可继续使用现有“指派司机/车辆 → Trip → 执行 → 签收”的流程；
  - Trip 详情中可通过已有接口间接看到关联 Waybill，从而间接追溯到转运单。

### 6. 与报价卡的集成（v0.1 最小要求）

> 说明：本期不强求完整实现复杂规则，只要求打好模型基础和最小的在线匹配能力。

#### 6.1 匹配维度（概念）

- 合作单位（JS/JW/ESC/无忧达等）；
- 仓号/所在仓库（YYZ3/YYZ4/YYZ7/YYZ9/YOO1/YHM1/YXU1/YOW1 等）；
- 托数区间（1–4 板、5–14 板、15–28 板、散板/拼货）；
- 运输类型（直卡/拖车/散板）；
- 是否私人地址、是否贴标、是否换板等增值服务选项。

#### 6.2 v0.1 要求

- 在转运单明细模型中预留上述匹配维度与价格字段；
- 在后端为“报价匹配”预留一个 API：
  - `POST /api/pricing/transfer/preview`（示例命名）：
    - 输入：单条明细的信息；
    - 输出：建议客户报价、建议成本价、匹配规则 ID 或说明；
  - 前端在编辑明细时可以调用该接口，实时展示建议价格；
- 若匹配失败：
  - 前端在行级显示明显提示（例如红色“未匹配价格规则”）；
  - 阻止直接生成运单，要求运营手工填入价格并备注来源。

### 7. 界面结构（概要）

#### 7.1 转运单列表页

- 列表字段：
  - 转运单号、柜号、所在仓库、到仓日、合作单位、主目的仓、状态、总托数/已生成托数/HOLD 托数、创建时间/人。
- 功能：
  - 筛选：柜号、仓库、目的仓、合作单位、状态、时间区间；
  - 动作：新建转运单、查看/编辑、批量生成运单（从多个转运单明细中分组）。

#### 7.2 转运单详情/编辑页

- 上半部分：转运单头表单。
- 中间：明细列表（表格），支持：
  - 行内编辑（SKU/FBA/件数/托数/方数/目的仓等）；
  - 行级操作：
    - 拆分明细；
    - 标记/解除 HOLD；
    - 查看/编辑价格；
    - 单行生成运单。
- 底部：批量操作区域：
  - 勾选多行 → 统一“生成运单”；
  - 显示合计托数/方数以及报价预估。

### 8. 异常与边界情况

- 未填写必填字段 → 禁止保存/生成运单，给出明确提示。
- HOLD 明细被误选加入生成运单 → 前端直接过滤/禁用勾选，不允许进入下一步。
- 一行明细托数过大需拆分 → 提供“拆分明细”功能，要求填入拆分后的托数，系统校验总和不变。
- 部分明细已生成运单，部分仍未生成 → 状态显示为 `PARTIAL_WAYBILLED`，并在明细列表中清晰标记已/未生成数量。
- 删除/取消转运单：
  - 若转运单有已生成运单，禁止彻底删除，只允许状态改为 `CANCELLED`，并确保不影响已生成运单履约。
